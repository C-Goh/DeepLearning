Diskussion der Ergebnisse

  1. Datenmenge: Mit wenigen hundert Wörtern lernt das Modell nur Basispatterns; ab 1.000+ Wörtern werden Vorhersagen deutlich konsistenter.
  2. Sequenzlänge: Längere Kontextfenster (7–10) verbessern die Qualität, erhöhen aber Speicher- und Zeitbedarf.
  3. Overfitting: Bei kleinen Datensätzen tendiert das Modell zum Memorieren; Dropout reduziert diesen Effekt.
  4. Top-K Accuracy: Top-1 liegt typisch bei 5–15 %, Top-10 oft bei 30–50 % – das Modell schlägt sinnvolle Kandidaten vor.
  5. Rekonstruktion & Datenschutz: Bei ausreichendem Training kann das Modell Trainingsdaten reproduzieren – sensible Daten sollten nie als Trainingskorpus verwendet werden.

Fazit: Zwei gestapelte LSTM-Layer erfassen kurz- und mittelfristige Abhängigkeiten gut. Für produktionsreife Anwendungen wären deutlich größere Datenmengen und Modelle erforderlich.

Architektur

  • Embedding Layer – Wort-Indizes → dichte Vektoren (default 64 Dim.)
  • LSTM Layer 1 – 100 Units, returnSequences = true
  • Dropout – 20 % Regularisierung
  • LSTM Layer 2 – 100 Units, returnSequences = false
  • Dropout – 20 % Regularisierung
  • Dense + Softmax – Ausgabe über gesamtes Vokabular

Besonderheiten

  • Explizites tf.dispose() nach jeder Inferenz (kein Memory Leak).
  • Epoch-weises Training (epochs: 1 im Loop) für unterbrechbares UI.
  • Ladeoverlay blockiert UI sauber während Modell-Kompilierung.

Fachliche Dokumentation

Language Model

Ein LM modelliert P(w_t | w_1,…,w_{t-1}). Das trainierte LSTM kann Text autoregressiv generieren, indem das wahrscheinlichste Wort an den Kontext angehängt wird.

LSTM

LSTMs lösen das Vanishing-Gradient-Problem klassischer RNNs durch Input-, Forget- und Output-Gates und sind dadurch besonders geeignet für sprachliche Langzeitabhängigkeiten.

Metriken

  • Top-K Accuracy – Anteil der Fälle, in denen das korrekte Wort unter den K wahrscheinlichsten liegt.
  • Perplexity – exp(Cross-Entropy-Loss); niedrigere Werte = besseres Modell.

Quellen

  • Mikolov et al. (2010): Recurrent neural network based language model
  • Hochreiter & Schmidhuber (1997): Long Short-Term Memory
  • TensorFlow.js Dokumentation

Hilfe & Schnellstart

  1. Beispieltext laden oder eigenen Text eingeben.
  2. „1. Daten vorverarbeiten" klicken → Vokabular wird aufgebaut.
  3. „2. Modell erstellen" klicken → LSTM-Architektur wird kompiliert.
  4. „3. Training starten" → Loss beobachten, bei Plateau stoppen.
  5. Im Abschnitt Vorhersage Starttext eingeben und Wörter generieren.

Tipps

  • Mindestens 500–1000 Wörter für spürbare Qualität.
  • NaN-Loss → Learning Rate auf 0.0001 reduzieren.
  • Browser hängt → Epochen/Batchgröße reduzieren oder Seite neu laden.
