Diskussion der Ergebnisse

Beobachtungen:

  1. Datenmenge: Die Qualität der Vorhersagen hängt stark von der Menge und Qualität der Trainingsdaten ab. Mit wenigen hundert Wörtern lernt das Modell nur grundlegende Muster; mit mehreren tausend Wörtern werden die Vorhersagen deutlich besser.
  2. Sequenzlänge: Längere Kontextfenster (7-10 Wörter) verbessern die Vorhersagequalität, erhöhen aber die Trainingszeit und den Speicherbedarf.
  3. Overfitting: Bei kleinen Datensätzen tendiert das Modell zum Auswendiglernen. Der Dropout hilft, dies zu reduzieren.
  4. Top-K Accuracy: Die Top-1 Accuracy liegt typischerweise bei 5-15%, aber die Top-10 Accuracy erreicht oft 30-50%, was zeigt, dass das Modell sinnvolle Kandidaten vorschlägt.
  5. Rekonstruktion: Das Modell kann bei ausreichendem Training Teile der Trainingsdaten rekonstruieren, was Datenschutzimplikationen hat.

Lessons Learned: Language Models lernen statistische Muster in Texten. Die Architektur mit zwei LSTM-Layern erfasst sowohl kurzfristige als auch längerfristige Abhängigkeiten. Für produktionsreife Anwendungen wären deutlich mehr Daten und größere Modelle nötig.

Architektur-Details

  • Embedding Layer: Wandelt Wort-Indizes in dichte Vektoren um (default: 64 Dimensionen)
  • LSTM Layer 1: Erste rekurrente Schicht mit returnSequences=true für den zweiten Layer
  • Dropout: Regularisierung zwischen LSTM-Layern (default: 20%)
  • LSTM Layer 2: Zweite rekurrente Schicht für komplexere Muster
  • Dense + Softmax: Ausgabeschicht mit Wahrscheinlichkeiten für jedes Wort im Vokabular

Datenvorverarbeitung

  1. Tokenisierung: Text wird in Wörter aufgeteilt (Leerzeichen-basiert)
  2. Lowercase-Normalisierung für konsistentes Vokabular
  3. Vokabular-Erstellung mit Wort-zu-Index Mapping
  4. Sequenz-Generierung: Sliding Window über den Text
  5. One-Hot-Encoding der Zielwörter für Cross-Entropy Loss

Besonderheiten der Implementierung

  • Memory Management: Explizites tf.dispose() für Tensor-Cleanup zur Vermeidung von Memory Leaks
  • Asynchrones Training: Verwendung von async/await für non-blocking UI
  • Dynamisches Batching: Anpassung der Batch-Größe basierend auf Datenmenge
  • Progressive Visualisierung: Echtzeit-Updates der Loss-Kurve während des Trainings

Fachliche Dokumentation

Language Model Konzept

Ein Language Model (LM) modelliert die Wahrscheinlichkeitsverteilung P(w_t | w_1, w_2, ..., w_{t-1}) über Wörter, gegeben den vorherigen Kontext. Das trainierte Modell kann verwendet werden, um das wahrscheinlichste nächste Wort vorherzusagen oder Text autoregressiv zu generieren.

LSTM Architektur

Long Short-Term Memory (LSTM) Netzwerke sind eine spezielle Art von RNNs, die das Vanishing Gradient Problem durch Gates (Input, Forget, Output) lösen. Sie können sowohl kurzfristige als auch langfristige Abhängigkeiten in Sequenzen lernen.

Evaluation Metriken

  • Top-K Accuracy: Prozentsatz der Fälle, in denen das korrekte Wort unter den K wahrscheinlichsten Vorhersagen ist
  • Perplexity: Exp(Cross-Entropy Loss) - misst, wie "überrascht" das Modell von den Testdaten ist. Niedrigere Werte sind besser.
  • Cross-Entropy Loss: Misst die Differenz zwischen vorhergesagter und tatsächlicher Wahrscheinlichkeitsverteilung

Datenschutz-Implikationen

Language Models können bei ausreichendem Training Teile ihrer Trainingsdaten "memorieren" und reproduzieren. Dies ist besonders problematisch, wenn sensible Daten (z.B. persönliche Informationen, Passwörter) im Training verwendet werden. Moderne Ansätze wie Differential Privacy können dieses Risiko reduzieren.

Hilfe & Anleitung

Schnellstart

  1. Wechseln Sie zum "Training" Tab
  2. Laden Sie einen Beispieltext oder geben Sie eigenen Text ein
  3. Klicken Sie "Daten vorverarbeiten"
  4. Klicken Sie "Modell erstellen"
  5. Klicken Sie "Training starten" und warten Sie, bis der Loss sinkt
  6. Wechseln Sie zum "Vorhersage" Tab und generieren Sie Text

Tipps für bessere Ergebnisse

  • Verwenden Sie mindestens 1000+ Wörter für das Training
  • Homogener Text (z.B. nur Nachrichtenartikel) führt zu konsistenteren Vorhersagen
  • Experimentieren Sie mit der Sequenzlänge (5-10 Wörter)
  • Beobachten Sie den Loss - wenn er nicht mehr sinkt, stoppen Sie das Training
  • Bei Overfitting: Mehr Dropout oder weniger Epochen

Fehlerbehebung

  • Browser friert ein: Reduzieren Sie die Textmenge oder Epochenzahl
  • NaN Loss: Reduzieren Sie die Learning Rate auf 0.0001
  • Schlechte Vorhersagen: Mehr Training oder mehr Daten
  • Memory Error: Aktualisieren Sie die Seite und verwenden Sie kleinere Batches
